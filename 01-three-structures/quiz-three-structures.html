如果 a 和 b 均为 int 类型的变量，下列表达式能正确判断"a等于0或b等于0"的是（ ）
A(!a) || (!b)
B(a == b == 0)
C(a == 0) && (b == 0)
D(a == 0) - (b == 0) == 0
✏️ 解析
逐一分析：
A. (!a)||(!b) → !a 在 a 为 0 时为真（1），为 0 时为假（0）；|| 表示"或"，只要 a 或 b 有一个为 0，结果就为真 ✓
B. (a==b==0) → C++ 链式比较：先算 a==b 得 0 或 1，再和 0 比较，无法正确判断 ✗
C. (a==0)&&(b==0) → && 是"且"，表示 a 和 b 都为 0 才成立，不符合"或"的要求 ✗
D. (a==0)-(b==0)==0 → 当 a=0 且 b=0 时结果为 0，但当 a=0、b≠0 时结果为 -1≠0，无法正确判断"或"的关系 ✗
💡 C++ 中 !a 是逻辑非运算符，对 int 类型变量，值为 0 时 !a 为 1（真），值非 0 时 !a 为 0（假）。这是 C++ 的标准用法。
✅ 答案：A · (!a)||(!b)

如果 a 为 char 类型的变量，下列哪个表达式可以正确判断"a是小写字母"？
Aa<=a<=z
Ba-'a'<='z'-'a'
C'a'<=a<='z'
Da>='a' && a<='z'
✏️ 解析
核心知识点：C++不支持数学中的链式比较。
A. a<=a<=z → 语法混乱 ✗
B. a-'a'<='z'-'a' → 当 a='A' 时，'A'-'a'为负数，负数<=25 仍为true ✗
C. 'a'<=a<='z' → 链式比较：先算 'a'<=a 得 0或1，再判断 0/1<='z' 始终为true ✗
D. a>='a' && a<='z' → 两个条件同时满足 ✓
⚠️ C 选项是最常见的陷阱！'a'<=a<='z' 在 Python 里正确，但在 C++ 里会被解析为 ('a'<=a) <= 'z'，几乎永远为 true。
✅ 答案：D · a>='a' && a<='z'

Bi == 4

用 ||（逻辑或）连接：i == 0 || i == 4

判断回文数，横线处应填写？（ ）
int n, a, k; cin >> n; a = 0; k = n; while (n > 0) { a = ________; // 在此处填写代码 n /= 10; } if (a == k) cout << "是回文数"; else cout << "不是回文数";
A10 * a + n % 10
Ba + n % 10
C10 * a + n / 10
Da + n / 10
✏️ 解析
算法思路：逐位取出 n 的末尾数字，构建逆序数 a。以 n=123 为例：
初始 a=0, n=123
第1轮：a = 10*0 + 123%10 = 3，n = 12
第2轮：a = 10*3 + 12%10 = 32，n = 1
第3轮：a = 10*32 + 1%10 = 321，n = 0，退出
10 * a 把已有数字左移一位（腾出个位），n % 10 取末位数字填入。
⚠️ B 缺少 *10，数字无法正确左移；C、D 用了 n/10 取的是去掉末位后的数，逻辑完全错误。
✅ 答案：A · 10 * a + n % 10

double s; int t; s = 18.5; t = int(s) + 10;
✓ 正确
✗ 错误

for(i = 0; i < 100; i += 2) ; 语句中变量 i 的取值范围是 0 到 99。
✓ 正确
✗ 错误
✏️ 解析
i 的取值为 0, 2, 4, 6, ..., 98（全是偶数）。
i 从 0 开始，每次 +2，条件 i < 100

"取值范围是0到99"有两处错误：

已知 A 的 ASCII 码值为 65，表达式 int('C') + abs(-5.8) 的值为 72.8。
✓ 正确
✗ 错误

20
任何一个 for 循环都可以转化为等价的 while 循环。（ ）
✓ 正确
✗ 错误
✏️ 解析
for 循环和 while 循环是等价的控制结构：
for(init; condition; update) { body }
等价于：
init; while(condition) { body; update; }
for(int i=0; i<10; i++) cout<<i; // 等价于 int i=0; while(i<10) { cout<<i; i++; }
这是编程语言的基本等价关系，任何 for 循环都能用 while 重写，反之亦然。
✅ 答案：正确
