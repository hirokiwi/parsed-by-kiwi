如果字符变量 _1 的值是字符 '1'，那么 (int)_1 的值是？
A1
B-1
C49
D+1 或者 -1
✏️ 解析
字符 '1' 在 ASCII 码表中的编码值是 49，不是数值 1。
关键区分：
字符 '1' → ASCII 值 49
数值 1 → 整数值 1
(int)_1 是将字符型转为整型，结果为该字符的 ASCII 编码值。
⚠️ 易错点：容易混淆字符 '1' 和数值 1。字符 '0' 的 ASCII 码是 48，'1' 是 49，'2' 是 50……以此类推。
✅ 答案：C · 49

a, b 是整型变量，各自有互不相同的初始值。下列程序实现了什么效果？
a = a ^ b; b = a ^ b; a = a ^ b;
Aa, b 的值从始至终都没有改变
Ba, b 的值实现了互换
Ca, b 的值互换了以后，又还回去了
Da, b 的值最后和原值不一样，没有任何意义
✏️ 解析
异或交换（XOR Swap）
设初始 a = A, b = B
第1步：a = a ^ b → a = A ^ B
第2步：b = a ^ b → b = (A ^ B) ^ B = A ^ (B ^ B) = A ^ 0 = A
第3步：a = a ^ b → a = (A ^ B) ^ A = (A ^ A) ^ B = 0 ^ B = B
最终：a = B, b = A ✅ 交换成功
💡 原理：异或运算满足交换律和结合律，且 x ^ x = 0，x ^ 0 = x。三次异或可以在不借助临时变量的情况下完成交换。
⚠️ 注意：若 a 和 b 指向同一内存地址，此方法会失败（a ^ a = 0），但题目说明"互不相同的初始值"，所以可以正常工作。
✅ 答案：B · a, b 的值实现了互换

关于下列正确的程序段，说法正确的是？
char str1[] = "Hello"; char str2[] = {'H','e','l','l','0'};
cout << str1 << endl; cout << str2 << endl;
A字符数组 str1 和 str2 完全相同
B这段程序多次执行将输出不同的结果
C字符数组 str1 和 str2 不相等
D这两种赋值方式完全相同
✏️ 解析
str1[] = "Hello"
→ {'H','e','l','l','o','\0'}，共6个元素，自动添加结束符 '\0'

str2[] = {'H','e','l','l','0'}
→ 共5个元素，没有自动添加 '\0'，且最后一个是数字字符 '0'(ASCII 48)，不是字母 'o'(ASCII 111)
⚠️ 双重陷阱！
1️⃣ str2 的最后一个字符是 '0'（数字零），不是 'o'（字母o），内容本身就不同
2️⃣ str1 用双引号初始化会自动追加 '\0'，str2 用花括号逐字符初始化不会追加，长度不同
3️⃣ cout << str2 因缺少 '\0' 是未定义行为，可能输出乱码
✅ 答案：C · str1 和 str2 不相等

关于以下程序段，说法正确的是？
int x = 10; cout << (x++) + (++x) << endl;
AC++11标准中，这是未定义行为，不同环境可能出现不同结果
B22
C21
D20
✏️ 解析
未定义行为（Undefined Behavior）
表达式 (x++) + (++x) 中，x 在同一个序列点内被修改了两次：
• x++ 先取值后自增（修改x）
• ++x 先自增（修改x）后取值

C++ 标准规定：在两个序列点之间，同一变量最多只能被修改一次。违反此规则即为未定义行为。
⚠️ 不要试图"算"出结果！不同编译器可能给出 20、21、22 或其他值，这取决于编译器的求值顺序实现，没有"正确"答案。GESP 考试中此类题目答案就是"未定义行为"。
💡 判断技巧：同一表达式中对同一变量既有前缀又有后缀自增/自减，直接判定为未定义行为。
✅ 答案：A · 未定义行为

8位二进制下，十进制数 -15 的补码是？
A11110000
B10001111
C10010000
D11110001
✏️ 解析
补码求法三步走
第1步：写出绝对值的二进制
15 = 00001111（8位）

第2步：按位取反（反码）
~00001111 = 11110000

第3步：加1（补码）
11110000 + 1 = 11110001
⚠️ 易错点：选项A 11110000 是反码，不是补码！别忘了最后一步"+1"。
选项B 10001111 是原码（符号位+绝对值），也不是补码。
✅ 答案：D · 11110001

A63
B65
C67
D69
✏️ 解析
进制转换：按权展开法
2102(3) = 2×3³ + 1×3² + 0×3¹ + 2×3⁰

= 2×27 + 1×9 + 0×3 + 2×1
= 54 + 9 + 0 + 2
= 65
💡 公式：N进制数 dndn-1...d1d0 = Σ di × Ni，从右到左权值依次为 N⁰, N¹, N², ...
✅ 答案：B · 65

已知 unsigned char c = 0x0F;（十六进制0F = 二进制00001111），执行 c = c << 3; 后，c 的十进制值是？
A64
B72
C80
D120
✏️ 解析
左移运算
初始值：0x0F = 00001111

左移3位：00001111 << 3 = 01111000

转十进制：
01111000 = 0×128 + 1×64 + 1×32 + 1×16 + 1×8 + 0×4 + 0×2 + 0×1
= 64 + 32 + 16 + 8 = 120
💡 快捷算法：左移n位 = 乘以 2ⁿ。15 << 3 = 15 × 8 = 120。
⚠️ unsigned char 只有8位，左移后超出部分会被丢弃，这里没有溢出。
✅ 答案：D · 120

补码的情况下，关于按位取反运算，以下说法错误的是？
A~5 的结果是 -6（int类型，32位）
B~0 的结果是 0（int类型，32位）
C~(-3) 的结果是 2（int类型，32位）
D~8 的结果是 -9（int类型，32位）
✏️ 解析
按位取反公式：~x = -(x+1)
补码下，按位取反的核心公式：~x = -(x+1)

验证各选项：
A. ~5 = -(5+1) = -6 ✅ 正确
B. ~0 = -(0+1) = -1 ≠ 0 ❌ 错误！
C. ~(-3) = -(-3+1) = -(-2) = 2 ✅ 正确
D. ~8 = -(8+1) = -9 ✅ 正确
⚠️ ~0 的结果是 -1，不是 0！0 的二进制全0，取反后全1，即 -1 的补码。这是最常见的考点。
✅ 答案：B · ~0的结果是0（此说法错误，实际结果为-1）

执行以下代码后，输出结果是？
int arr[] = {5, 10, 15, 20, 25, 30}; int count = sizeof(arr) / sizeof(arr[0]); cout << count;
A4
B6
C24
D30
✏️ 解析
sizeof 计算数组元素个数
sizeof(arr) = 数组总字节数 = 6 × 4 = 24（每个int占4字节）
sizeof(arr[0]) = 单个元素字节数 = 4

这是C/C++中计算数组长度的标准写法。
⚠️ 选项C的24是数组总字节数，选项D的30是最后一个元素的值，都不是答案。
✅ 答案：B · 6

执行以下代码后，输出结果是？
char s[10] = "abcde"; int a = sizeof(s) / sizeof(s[0]); int b = strlen(s); cout << a - b;
A5
B6
C10
D0
✏️ 解析
sizeof vs strlen
char s[10] = "abcde";
→ 数组大小为10，内容：'a','b','c','d','e','\0',0,0,0,0

sizeof(s) = 10（数组声明的总大小）
sizeof(s[0]) = 1（char占1字节）
a = 10 / 1 = 10

strlen(s) = 5（到 '\0' 为止的字符数，不含 '\0'）

a - b = 10 - 5 = 5
💡 关键区别：
• sizeof：编译时确定，返回数组声明的大小（含未使用的空间）
• strlen：运行时计算，返回第一个 '\0' 前的字符数
本题中 10 - 5 = 5，差值正好是"未使用空间 + '\0'"的个数。
✅ 答案：A · 5

以下问题中，最不适合用枚举法解决的是？
A找出1~100之间所有能被7整除的数
B找出100~200之间的所有质数
C计算1+2+3+…+1000的和
D找出三位数中个位、十位、百位数字之和等于10的数
✏️ 解析
枚举法的适用场景
枚举法适用于搜索/筛选类问题，即"找满足条件的元素"。
不适用于有高效公式直接计算的问题。
A. 找能被7整除的数 → 逐个判断，适合枚举
B. 找质数 → 需要逐个判断，适合枚举
C. 求和 1+2+…+1000 → 有公式 n×(n+1)/2 = 500500，一步算出，无需枚举
D. 找各位数字之和等于10 → 需要逐个检查，适合枚举
💡 判断标准：如果能用数学公式直接求解，就不应该用枚举法。枚举法的核心是"穷举所有可能再筛选"，效率低于公式法。
✅ 答案：C · 计算求和有公式，无需枚举

用枚举法解决"鸡兔同笼：头共35个，脚共94只"，以下枚举逻辑最合理的是？
A枚举鸡的数量x（0~35），兔y=35-x，判断 2×x + 4×y == 94
B枚举兔的数量y（0~94），鸡x=35-y，判断 2×x + 4×y == 94
C枚举所有整数x（0~100）和y（0~100），判断 x+y==35 && 2×x+4×y==94
D枚举脚的总数sum（0~94），判断 sum == 94
✏️ 解析
枚举法的优化策略
A：枚举x（0~35），y由约束条件 y=35-x 自动确定，只需循环36次 ✅

B：兔的范围应为0~35（受头数约束），0~94范围过大且x=35-y不合理（y可能>35）❌

C：双重循环101×101=10201次，完全没利用头数约束，效率低 ❌

D：枚举脚的总数毫无意义，脚的总数就是94 ❌
💡 枚举优化原则：充分利用已知约束减少枚举变量和范围。
本题"头=35"是硬约束，确定一个变量后另一个自动确定，只需单层循环。
✅ 答案：A · 枚举鸡的数量，兔由约束确定

模拟"字符串加密"：规则为"每个字符ASCII码+3，若超过z(122)则从a重新开始"，以下代码中正确的条件判断是？
for (int i = 0; i < str.length(); i++) { // 需补充条件 _______________ else { str[i] += 3; } }
Aif (str[i] > 122) str[i] = str[i] + 3 - 26;
Bif (str[i] > 122) str[i] = str[i] - 26;
Cif (str[i] + 3 > 122) str[i] = str[i] + 3 - 26;
Dif (str[i] + 3 > 'z') str[i] = 'a' + (str[i] + 3 - 'z' - 1);
✏️ 解析
字符串加密——循环移位
加密规则：字符+3，超过'z'(122)则从'a'循环

A 错：条件 str[i] > 122 判断的是原字符是否超过122，小写字母最大就是122('z')，条件永远为false

B 错：同A条件错误，且没有+3操作

C 看似正确但有隐患：str[i] = str[i] + 3 - 26 在char类型下可能溢出。str[i]+3先提升为int运算，但赋值回char时需要强转。更关键的是，这种写法逻辑上虽然数学等价，但不如D直观

D 正确：判断加3后是否超过'z'，若超过则从'a'开始偏移 (str[i]+3-'z'-1) 位
验证D：以'z'(122)为例
条件：122+3=125 > 'z'(122) → true
执行：'a' + (122+3-122-1) = 97 + 2 = 99 = 'c' ✅

验证D：以'w'(119)为例
条件：119+3=122，不大于'z' → else分支
执行：str[i] += 3 → 119+3=122='z' ✅
💡 D 的公式本质：'a' + (新位置 - 'z' - 1) 表示超出'z'的部分从'a'重新计数，这是凯撒密码（Caesar Cipher）循环移位的标准写法。
✅ 答案：D
