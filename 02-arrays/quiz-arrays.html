排序算法的稳定性：待排序数据中 R 和 S 相等且 R 在 S 前，排序后 R 仍在 S 前。下面关于排序稳定性的描述，正确的是（ ）
A冒泡排序是不稳定的
B插入排序是不稳定的
C选择排序是不稳定的
D以上都不正确
✏️ 解析
排序算法稳定性总结
冒泡排序：只交换相邻逆序对，相等不交换 → 稳定
插入排序：找到位置后插入，相等元素后到的排后面 → 稳定
选择排序：每轮选最小值交换，可能跨过相等元素 → 不稳定
⚠️ 选择排序不稳定举例：
数组 [5a, 5b, 2] → 第一轮选最小值 2 与 5a 交换 → [2, 5b, 5a]
5a 原在 5b 前，交换后 5b 在 5a 前 → 不稳定。
✅ 答案：C · 选择排序是不稳定的

冒泡排序的第一轮操作是从左到右遍历数组，通过两两比较相邻元素，将当前最大的元素移动到末尾。给定数组 arr[]={4, 1, 3, 1, 5, 2}，执行第一轮冒泡排序后数组 arr 中的内容为（ ）
A1, 4, 3, 1, 5, 2

C1, 4, 3, 1, 2, 5

💡 第一轮结束，最大值 5 沉到底部。A 选项只比了第一步就停了。

小杨在整理一副扑克牌的所有红心扑克牌，使其从小到大排列。他的做法是：最开始抓到第1张扑克牌被认为已经排好序；然后抓第2张扑克牌，将其插入至有序部分的正确位置；不断循环步骤，每次将新抓到扑克牌插入至有序部分，直至抓完所有扑克牌，这样抓牌结束时就完成了扑克牌的排序。小杨这种整理扑克牌的方式与（ ）排序的方式最接近。
A冒泡排序
B插入排序
C选择排序
D直接排序
✏️ 解析
题目描述的每一步都是"将新元素插入到已排序部分的正确位置"——这正是插入排序的核心思想。
冒泡排序：相邻元素两两比较交换，大数逐步"冒泡"到末尾
插入排序：逐个取出元素，插入到前面已排序序列的正确位置 ✓
选择排序：每轮从未排序部分选出最小值放到已排序部分末尾
💡 D 选项"直接排序"不是标准排序算法名称。
抓牌插入 = 插入排序的经典生活类比。
✅ 答案：B · 插入排序
