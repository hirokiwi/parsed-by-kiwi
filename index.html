C++ 指针 与 引用

指针是存储地址的变量，引用是变量的别名。两者都能间接操作数据，但机制和用法截然不同。通过可视化动画，彻底搞懂它们！

指针基础 取地址 & 解引用 引用本质 传参对比 常见陷阱
§1 指针基础

指针是什么？

指针是一种特殊变量，它存储的不是普通数据，而是另一个变量的内存地址。就像一张"地址便签"，通过它可以间接找到并操作目标变量。

📍指针的声明与定义

pointer_basic.cpp
1#include <iostream>
2using namespace std;
3
4int a = 42; // 定义一个整型变量 a
5int * p = &a; // 定义指针 p，指向 a 的地址
6
7cout << "a = " << a << endl; // 直接访问 a
8cout << "*p = " << *p << endl; // 通过指针 p 访问 a（解引用）
9cout << "p = " << p << endl; // 打印 p 存储的地址

• 指针本身也占用内存，也有自己的地址
§2 取地址 & 解引用

两个核心运算符

& 取地址 和 * 解引用，是指针操作的灵魂。

&取地址运算符 &

用在变量前，获取该变量的内存地址。

1int x = 100;
2int *px = &x; // px 获得 x 的地址
x → 0x7ffd...
&x 的结果是一个地址值

*解引用运算符 *

用在指针前，通过地址找到并访问指向的变量。

1int x = 100;
2int *px = &x;
3cout << *px; // *px 读 x 的值 → 100
4*px = 200; // 通过 px 修改 x → x = 200
*px → x → 100
*px 找到 px 存的地址，对应变量 x

➕指针算术运算

指针加 1，不是地址值 +1，而是指向"下一个同类型元素"。

int arr[5] = {10,20,30};
int* p = arr;
arr == &arr[0]
p+1 == &arr[1] （地址 +4字节）
*(p+1) == 20
指针 + n
向后移动 n 个"类型单位"。
int*: +n → 地址 + n×4 字节
double*: +n → 地址 + n×8 字节
*p++ vs *(p++)
*p++：先取 *p 的值，
再把指针向后移 1 位。
常用于遍历数组。
§3 引用

变量的别名

引用不是独立变量，而是已有变量的另一个名字。一旦绑定，终身不变。就像给一个人取了个外号——无论你叫他本名还是外号，都是同一个人。

🔗引用的基本用法

reference_basic.cpp
1int a = 10; // 变量 a
2int & r = a; // r 是 a 的引用（别名）
3
4r = 99; // 通过 r 修改，等价于 a = 99
5cout << a; // 输出 99
6
7cout << &r; // &r 的地址 == &a（同一个位置）
💡
引用与原变量共用同一块内存，修改引用就是修改原变量，&r 和 &a 的地址完全相同！

§4 三种传参方式对比

交换函数的正确打开方式

这是信奥一本通的重点内容！同一个"交换两数"需求，用三种不同的参数传递方式，效果截然不同。

✗值传递（传的是副本）

1void swap(int x, int y) { // 形参是副本
2 int t = x; x = y; y = t;
3}
4int a=1, b=2;
5swap(a, b); // 实参不受影响
6cout << a << ' ' << b; // 输出 1 2（未交换！）
❌ 交换失败：a=1, b=2（a 和 b 的值没变）
🧠
原因：形参 x、y 是 a、b 的副本。函数内交换的只是副本，原变量纹丝不动。就像复印了一份合同去修改，原件当然不变。
栈帧示意图
main 栈帧
变量
a = 1
b = 2
swap 栈帧
形参副本
x = 1 → 2
y = 2 → 1
⚠ swap 函数结束后，swap 栈帧被销毁，交换结果丢失

✓指针传参（传地址）

1void swap(int *x, int *y) {
2 int t = *x;
3 *x = *y;
4 *y = t;
5}
6int a=1, b=2;
7swap(&a, &b); // 传地址
8cout << a << ' ' << b; // 输出 2 1 ✓
✅ 交换成功：a=2, b=1
💡
原理：通过 &a 把 a 的地址传给指针形参，函数内用 *x 找到 a 的位置直接修改——就像拿到钥匙🔑去房间改东西。

✓引用传参（最简洁）

1void swap(int & x, int & y) {
2 int t = x; x = y; y = t;
3}
4int a=1, b=2;
5swap(a, b); // 直接传 a, b，像值传递一样简洁
6cout << a << ' ' << b; // 输出 2 1 ✓
✅ 交换成功：a=2, b=1
✨
C++ 的优势：引用传参语法上和值传递一样简洁（不需要写 & 和 *），但效果上却能直接修改实参。这是 C++ 优于 C 语言的核心特性之一。
§5 对比总结

指针 vs 引用 · 十维对比

对比维度指针 *引用 &
定义方式int* pint& r = a
是否必须初始化可以不初始化（危险⚠）必须初始化（语法强制）
能否为空可以 nullptr不能为 null
能否重新指向可以 p = &b;不可以，r=b 是赋值操作
解引用需要 *p = 10直接使用 r = 10
sizeof 结果固定 4/8 字节与原变量相同（如 sizeof(int&)=4）
多级形式支持 int**、int***不支持（int&& 是右值引用）
函数参数检查需要检查是否为 nullptr无需检查（默认有效）
底层本质独立变量，存地址值原变量的别名（底层是常量指针）
语法简洁度需要 & 和 *，相对繁琐像普通变量一样使用，最简洁
🎯
实战原则：
• 能在任何场景下用引用就别用指针——引用更安全、更简洁
• 需要"不指向任何对象"时用指针（传参前记得判空）
• 需要动态改变指向目标时必须用指针
• C++ 中引用可以替代指针的绝大多数场景
§6 常见错误

新手必避的坑

💀 野指针（Wild Pointer）

指针声明后未初始化，就直接解引用访问，指向随机地址，可能导致程序崩溃或数据损坏。

1int *p; // ⚠ 未初始化，p 指向随机地址
2*p = 100; // 💀 危险！可能崩溃或破坏数据

✅ 正确：int* p = nullptr; 或 int* p = &a;

💀 空指针解引用（Null Dereference）

指针值为 nullptr 时解引用，程序直接崩溃（段错误）。

1int *p = nullptr;
2cout << *p; // 💀 崩溃！不能解引用 nullptr

✅ 正确：解引用前先判断 if(p != nullptr)

💀 悬空引用（Dangling Reference）

引用指向了一个已被销毁的变量（内存已被回收）。

1int& foo() {
2 int x = 10;
3 return x; // ⚠ x 在函数结束时销毁
4}
5int& r = foo(); // 💀 r 成了悬空引用
💀 对引用的"重新绑定"误解

误以为 ref = b 可以让 ref 指向 b，实际上只是把 b 的值赋给原变量。

1int a=10, b=20;
2int& r = a; // r 绑定到 a
3r = b; // ⚠ 不是"指向 b"，而是 a = b
4// 结果：a = 20, r 仍然绑定 a
