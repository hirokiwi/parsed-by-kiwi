C++表达式 (3 - 2) * 3 + 5 的值是（ ）
A-13
B8
C2
D0
✓ 答案 B

解析

按运算符优先级，先括号再乘法最后加法：
(3 - 2) = 1 1 * 3 = 3 3 + 5 = 8

执行C++语句 cin >> a 时如果输入 5+2，下述说法正确的是（ ）
A变量a将被赋值为整数7
B变量a将被赋值为字符串，内容为5+2

6
下面C++代码执行时输入21后，有关描述正确的是（ ）
int N; cin >> N; if(N % 3 == 0) cout << "能被3整除"; // 第4行 else if (N % 7 == 0) cout << "能被7整除"; // 第7行 else cout << "不能被3和7整除"; // 第9行
A代码第4行被执行
B第4和第7行代码都被执行
C仅有代码第7行被执行
D第8行代码将被执行，因为input()输入为字符串
✓ 答案 A

解析

输入 N = 21：
21 % 3 == 0 → true，进入 if 分支 执行第4行，输出 "能被3整除" else if 和 else 分支被跳过，不执行
关键：if-else if-else 是互斥分支，只会执行第一个满足条件的分支。21 既能被3整除也能被7整除，但因为先判断了 %3==0，就不再往下走了。
D 选项的 input() 是 Python 函数，与 C++ 无关。

Adouble类型的变量占用内存的大小是浮动的
Bbool类型的变量占用1字节内存

11
如果 a 和 b 均为 int 类型的变量，下列表达式能正确判断"a等于0或b等于0"的是（ ）
A(a == 0) || (b == 0)
B(a == 0) && (b == 0)
C(!a) || (!b)
D(a == b == 0)
✓ 答案 A

解析

逐一分析：
A. (a==0)||(b==0) → 用 || 表示"或"，语义明确 ✓ B. (a==0)&&(b==0) → && 是"且"，表示两个都为0 ✗ C. (!a)||(!b) → 对于 int，!a 等价于 a==0，逻辑上等价但用了隐式转换，不够规范 D. (a==b==0) → C++链式比较：先算 a==b 得 bool(0或1)，再和0比较，完全错误 ✗
C 选项虽然逻辑等价，但 !a 依赖隐式 bool 转换，可读性差，A 是最标准的写法。
12
如果 a 为 char 类型的变量，下列哪个表达式可以正确判断"a是小写字母"？
Aa<=a<=z
Ba-'a'<='z'-'a'
C'a'<=a<='z'
Da>='a' && a<='z'
✓ 答案 D

解析

核心知识点：C++不支持数学中的链式比较。
A. a<=a<=z → 语法混乱，变量a和标识符z混用 ✗ B. a-'a'<='z'-'a' → 当 a='A' 时，'A'-'a'为负数，负数<=25 仍为true ✗ C. 'a'<=a<='z' → 链式比较：先算 'a'<=a 得 0或1，再判断 0/1<='z' 始终为true ✗ D. a>='a' && a<='z' → 两个条件同时满足，才是小写字母 ✓
C 选项是最常见的陷阱！'a'<=a<='z' 在 Python 里正确，但在 C++ 里会被解析为 ('a'<=a) <= 'z'，几乎永远为 true。

在下列代码的横线处填写（ ），使得输出是 50 10
int a = 10, b = 50; ________; // 在此处填写代码 b -= a; a += b; cout << a << " " << b;
Aa -= b;
Ba += b;
Ca = b - a;
Da = b;
✓ 答案 C

解析

初始：a=10, b=50，目标输出 a=50, b=10（交换）
C. a = b - a → a = 50-10 = 40 然后 b -= a → b = 50-40 = 10 然后 a += b → a = 40+10 = 50 输出 50 10 ✓
其他选项验证：
A. a-=b → a=-40, b-=(-40)=90, a+=90=50 → 输出 50 90 ✗
B. a+=b → a=60, b-=60=-10, a+=(-10)=50 → 输出 50 -10 ✗
D. a=b → a=50, b-=50=0, a+=0=50 → 输出 50 0 ✗

"三天打渔，两天晒网"。小杨前三天打渔，后两天晒网，一直重复。以下代码判断第n天是打鱼还是晒网，横线处应填写？（ ）
int n, i; cin >> n; i = n % 5; if (________) // 在此处填写代码 cout << "晒网"; else cout << "打鱼";
Ai == 0
Bi == 4
Ci == 0 && i == 4
Di == 0 || i == 4
✓ 答案 D

解析

5天一周期：第1,2,3天打鱼，第4,5天晒网。
n % 5 的值：1,2,3 → 打鱼；4,0 → 晒网 晒网条件：i == 4（第4天）或 i == 0（第5天） 用 ||（逻辑或）连接：i == 0 || i == 4
选项 C 用 &&（逻辑与），i 不可能同时等于0和4，永远为 false。
验证：n=5 → i=0 → 晒网 ✓；n=4 → i=4 → 晒网 ✓；n=3 → i=3 → 打鱼 ✓
15
判断回文数，横线处应填写？（ ）
int n, a, k; cin >> n; a = 0; k = n; while (n > 0) { a = ________; // 在此处填写代码 n /= 10; } if (a == k) cout << "是回文数"; else cout << "不是回文数";
A10 * a + n % 10
Ba + n % 10
C10 * a + n / 10
Da + n / 10
✓ 答案 A

解析

算法思路：逐位取出 n 的末尾数字，构建逆序数 a。
以 n=123 为例： 初始 a=0, n=123 第1轮：a = 10*0 + 123%10 = 3，n = 12 第2轮：a = 10*3 + 12%10 = 32，n = 1 第3轮：a = 10*32 + 1%10 = 321，n = 0，退出
10 * a 把已有数字左移一位（腾出个位），n % 10 取末位数字填入。
B 缺少 *10，数字无法正确左移；C、D 用了 n/10 取的是去掉末位后的数，逻辑完全错误。

16
如果有以下C++代码，那么 cout << t 的结果为 28.5。
double s; int t; s = 18.5; t = int(s) + 10;
✓ 正确
✗ 错误
✓ 答案：错误

解析

int(s) = int(18.5) = 18（截断小数部分） t = 18 + 10 = 28 t 是 int 类型，cout << t 输出 28，不是 28.5
两个要点：
1. int(s) 截断（向零取整），不是四舍五入
2. 即使计算过程有小数，赋值给 int 变量后小数部分丢失
17
cout << (8 < 9 < 10) 的输出结果为 true。
✓ 正确
✗ 错误
✓ 答案：错误

解析

C++ 中链式比较从左到右：
8 < 9 → true（即 1） 1 < 10 → true（即 1） cout << 1 输出 1，而不是字符串 "true"
虽然表达式结果为 true，但 cout 默认输出 bool 值为 0/1，不是 "true"/"false"。
要输出 "true" 需使用 cout << boolalpha << (8<9<10)。
另外注意：这个链式比较的数学含义（8<9 且 9<10）在C++中并不成立，C++只是恰好算出了 true。
18
for(i = 0; i < 100; i += 2) ; 语句中变量 i 的取值范围是 0 到 99。
✓ 正确
✗ 错误
✓ 答案：错误

解析

i 的取值为 0, 2, 4, 6, ..., 98（全是偶数）。
i 从 0 开始，每次 +2，条件 i < 100 i 的最后一个值是 98，然后 i+=2 变为 100，退出循环
"取值范围是0到99"有两处错误：
1. i 只取偶数，不可能取到 1, 3, 5, ..., 99
2. 循环结束后 i = 100，超出 0~99 的范围

✓ 正确
✗ 错误
✓ 答案：正确

20
任何一个 for 循环都可以转化为等价的 while 循环。（ ）
✓ 正确
✗ 错误
✓ 答案：正确

解析

for 循环和 while 循环是等价的控制结构：
for(init; condition; update) { body } 等价于： init; while(condition) { body; update; }
例如：
for(int i=0; i<10; i++) cout<<i; // 等价于 int i=0; while(i<10) { cout<<i; i++; }
这是编程语言的基本等价关系，任何 for 循环都能用 while 重写，反之亦然。
